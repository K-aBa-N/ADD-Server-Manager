# ADD Server Manager

**ADD Server Manager** — приложение для управления несколькими серверами **DayZ** на Windows. Оно объединяет установку через SteamCMD, запуск процессов, мониторинг, управление рестартами, команды RCon, работа с модами из Workshop и веб-панель администратора для удобного управления.

## Назначение

Приложение решает следующие задачи для управления DayZ-серверами:

- создание и хранение **нескольких серверов** (инстансов) на одной машине;
- установка и обновление **игры и модов** через Steam Workshop;
- **старт / остановка / перезапуск** процессов `DayZServer_x64.exe`;
- мониторинг состояния, зависаний и крашей;
- управление **игроками, банами, whitelist** и конфигами через браузер;
- **расписание** рестартов и сообщений в чат;
- уведомления в **Discord / Telegram**.

Целевая ОС — **Windows**.

## Архитектура

| Слой | Технологии | Роль |
|------|------------|------|
| **Backend** | Node.js, Express 5, TypeScript, sql.js | API, процессы, Steam, RCon, планировщик |
| **Frontend** | Angular 21, Angular Material, Leaflet | SPA, отдаётся из `backend/public` |
| **Shared** | `shared/src` | Общие типы (инстанс, мод, WebSocket) |
| **Сборка** | `pkg` | Один исполняемый файл `DayzManager.exe` |

Веб-сервер слушает **только localhost** (`127.0.0.1`). Порт задаётся в `settings.cfg` (по умолчанию **8081**). На том же HTTP-сервере работает **WebSocket** для обновления статусов в реальном времени.

## Первый запуск

1. При отсутствии `settings.cfg` создаётся файл с настройками по умолчанию и открывается в Notepad — нужно заполнить параметры и перезапустить приложение.
2. Обязательно указать **`Steam.Username`** (и при необходимости пароль / API-ключ) для загрузки модов из Workshop. (API ключь требуется для поиска модов по их названию, если у вас нет возможности создать его используйте поиск по ID мода)
3. **Installer** проверяет системные зависимости, наличие `steamcmd.exe` и установленного DayZ Server; недостающее компоненты устанавливаются автоматически.
4. После успешной инициализации в логе появляется адрес веб-интерфейса, например: `http://localhost:8081`.

## Жизненный цикл приложения

При каждом запуске:

1. Загрузка `settings.cfg`, настройка логирования и языка (i18n).
2. Проверка и при необходимости установка компонентов (SteamCMD, DayZ Server).
3. Инициализация SQLite-базы `data/manager.db`.
4. Регистрация сервисов в DI-контейнере.
5. Загрузка инстансов из БД.
6. Опциональная проверка обновлений модов при старте.
7. Запуск периодической проверки обновлений **игры** (если включено).
8. Старт веб-сервера, сборщика метрик, **Monitor**, **Scheduler**.
9. **Автостарт** инстансов с флагом `autostart`.

При завершении (SIGINT / SIGTERM):

- остановка метрик и всех запущенных серверов;
- отключение RCon-соединений;
- остановка веб-сервера и сохранение БД.

## Функциональность

### Установка и Steam

- Скачивание и обновление **SteamCMD** и **DayZ Dedicated Server**.
- Загрузка модов Workshop (батчами, очередь без параллельных конфликтов SteamCMD).
- Автообновление сервера и модов по интервалам из `settings.cfg`.
- При обновлении мода или игры — **RestartCoordinator**: предупреждения в чат, кик, блокировка сервера (`#lock`), отложенный рестарт; объединение с запланированным рестартом при пересечении планов перезапуска.

### Инстансы

Каждый инстанс — отдельная конфигурация: игровой порт, query-порт, RCon, пароль RCon, `launchParams`, приоритет процесса, affinity, автостарт.

- создание, изменение, удаление;
- **start / stop / restart** (включая технический рестарт);
- **lock / unlock**, **kick-all**;
- импорт **существующего** сервера (скан папки, валидация портов, докачка модов);
- синхронизация `mods.json` и файлов модов на диске инстанса.

### Моды

- центральная библиотека модов по Workshop ID;
- установка, удаление, проверка хешей, массовое обновление;
- привязка к инстансам: порядок загрузки, server/client mod, включение/отключение.

### Мониторинг

- опрос через **gamedig** (игроки, отклик);
- метрики CPU/RAM процесса;
- **обнаружение зависания** (низкий CPU + нет ответа) → принудительный рестарт;
- **автоперезапуск** после краша процесса;
- ротация логов приложения.

### RCon (BattlEye)

- команды, глобальные сообщения, список игроков;
- кик, бан, сообщение игроку;
- интеграция с расписанием (предупреждения и кик перед рестартом).

### Планировщик

На уровне инстанса (cron):

- периодические **рестарты** с предупреждениями;
- **запланированные сообщения** в чат;
- пропуск следующего рестарта.

Таймзона берётся из глобальных настроек (`Local`, смещение UTC или IANA, например `Europe/Moscow`).

### Игроки

- онлайн через RCon, определение страны по IP;
- история заходов (first/last seen, количество подключений);
- баны и whitelist (файлы BattlEye), синхронизация банов между инстансами.

### Прочее

- парсинг и сохранение **serverDZ.cfg** (worker для тяжёлого парсинга);
- **метрики** в БД и API для графиков;
- **webhooks** — Discord / Telegram (старт, стоп, краш, рестарт, обновление мода);
- **Workshop API** — поиск и метаданные модов;
- просмотр логов, открытие папки инстанса в проводнике.

## Конфигурация

Основной файл — **`settings.cfg`** (INI-подобный формат).

| Секция | Назначение |
|--------|------------|
| `General` | Порт веб-UI, язык, уровень логов, таймзона |
| `Steam` | Учётные данные для Workshop |
| `Updates` | Интервалы и флаги автообновления, задержки и тексты перед рестартом при обновлении мода |
| `Process` | Graceful shutdown, порог «низкого CPU», elevate, affinity |
| `Logrotate` | Ротация логов приложения |
| `Paths` | `servers`, `steamcmd`, `logs` |
| `Messages` | Шаблоны сообщений (`{minutes}`, `{modName}`, `{version}`) |
| `Server` | Таймауты, hang detection, автоперезапуск после краша |
| `SteamCMD` | Таймауты, ретраи, размер батча модов |

Настройки отдельного инстанса (автостарт, webhooks, cron-рестарты) хранятся в БД и в JSON-конфиге инстанса.

## Данные на диске

| Путь | Содержимое |
|------|------------|
| `data/manager.db` | Инстансы, моды, метрики, игроки |
| `servers/<id>/` | Папка инстанса: `mods.json`, профили, логи |
| `steamcmd/` | SteamCMD, DayZ Server, Workshop-контент |
| `logs/` | Логи менеджера |
| `versions/` | Хранение скачанных версий игры |

Пример `mods.json` в папке инстанса — список модов с `modId`, порядком загрузки, флагами `enabled` и `isServerMod`.

## API

Префикс REST API: `/api/`

| Маршрут | Назначение |
|---------|------------|
| `/api/instances` | CRUD инстансов, старт/стоп, конфиг, моды, игроки, баны, логи, метрики |
| `/api/mods` | Глобальный каталог модов, скачивание, обновления |
| `/api/config` | Чтение/запись `settings.cfg` |
| `/api/rcon` | Команды RCon |
| `/api/scheduler` | Расписание рестартов и сообщений |
| `/api/metrics` | Метрики инстансов |
| `/api/workshop` | Поиск и информация о модах Steam |
| `/api/health` | Проверка работоспособности |
| `/version` | Версия приложения (текст) |

WebSocket: события статусов инстансов и прогресса операций.
